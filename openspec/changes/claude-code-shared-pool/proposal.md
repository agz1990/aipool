## Why

中国开发团队无法直接访问 Claude Code，需要通过美国服务器远程使用。如果每个团队成员都购买独立订阅，成本过高（15人团队每月$300）。通过共享池子方案，可以将成本降低73%（每月$80），同时支持团队错峰使用。

## What Changes

- 在美国开发服务器上部署 Claude Code 共享池子系统
- 创建多个共享账号（池子），每个池子登录独立的 Claude Code 订阅
- 实现池子占用/释放的协调机制，避免冲突
- 提供完整的管理工具（状态查看、监控、统计）
- 编写用户和管理员文档，确保团队能够正确使用

## Capabilities

### New Capabilities

- `pool-management`: 池子账号的创建、配置和管理，包括共享目录设置和软链接
- `coordination-system`: 池子占用/释放的协调机制，包括状态文件管理和锁定机制
- `user-tools`: 用户使用的命令行工具（查看状态、占用池子、释放池子、自动分配）
- `admin-tools`: 管理员使用的监控和维护工具（监控、报告、告警、备份）
- `deployment-automation`: 一键部署脚本，自动化整个系统的安装和配置
- `documentation`: 完整的用户指南、管理员手册和成本分析文档

## Impact

- **新增系统组件**: 在开发服务器上新增池子账号、共享目录、协调脚本
- **用户工作流变化**: 团队成员需要通过协调机制使用 Claude Code，而非直接登录
- **成本影响**: 大幅降低订阅成本（节省60-75%）
- **合规风险**: 账号共享可能违反 Anthropic 服务条款，需要准备备选方案
- **依赖关系**: 依赖美国服务器、Claude Code CLI、稳定的网络连接
